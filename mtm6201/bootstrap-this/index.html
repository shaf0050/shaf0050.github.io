Layout

Containers

Containers are a basic Bootstrap component that contain, pad, and align your content into a certain device or viewport.

How they work

Containers are the most fundamental layout element in Bootstrap, and they're necessary if you want to use our default grid system. Containers are used to hold, cushion, and (sometimes) centre the material they contain. Although containers can be nested, nested containers are not required in most layouts.


Default Container

The default .container class is a responsive, fixed-width container, which means its maximum width varies at each breakpoint.

Components

Navigation Bar

The navbar, Bootstrap's strong, responsive navigation header, has documentation and examples. Includes branding, navigation, and other features, as well as compatibility for our collapse plugin.

Before getting started with the navbar

For responsive collapsing and colour scheme classes, navbars require a wrapping.navbar with.navbar-expand-sm|-md|-lg|-xl|-xxl.


By default, navbars and their contents are fluid. In different methods, change the container to limit their horizontal width.


To adjust the spacing and alignment of navbars, use our spacing and flex utility classes.


Navbars are responsive by default, but you may alter that with ease. Our Collapse JavaScript plugin is responsible for responsive behaviour.


Use a nav> element to ensure accessibility, or add a role="navigation" to every navbar to clearly designate it as a landmark region for assistive technology users if using a more general element like a div>.

With support for various sizes, states, and more, use Bootstrap's custom button designs for actions in forms, dialogues, and more.


For Example

Bootstrap comes with a number of predefined button styles, each with its own semantic purpose and a few extras for further control.

Block Buttons

Using a combination of our display and gap tools, create responsive stacks of full-width, "block buttons" like those in Bootstrap 4. We have far more control over spacing, alignment, and responsive behaviours when we use utilities instead of button particular classes.
